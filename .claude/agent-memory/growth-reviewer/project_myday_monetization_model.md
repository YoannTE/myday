Suggestion initiale de structure tarifaire pour MyDay (dashboard unifié
agenda/mails/messages/IA), proposée lors de la revue de positionnement du
2026-07-09. Pas encore validée par l'utilisateur — à re-challenger si un
PRD/spec de pricing est produit plus tard.

**Tiers proposés** :
- Découverte (gratuit) : agenda + to-do uniquement, brief IA limité (ex. 1/semaine). Complet dans sa catégorie plutôt que rogné arbitrairement (cf. principe qualitatif > quantitatif).
- Essentiel (~19-24$/mois) : + mails + brief IA quotidien illimité + assistant conversationnel. Ancrage proche de Sunsama (20$/mois).
- Cercle (~35-40$/mois, jusqu'à 4-5 comptes liés) : + messageries (WhatsApp/SMS) + planning partagé famille. Tier qui porte la marge car il capitalise sur le différenciateur unique multi-comptes (cf. [[project_myday_market_positioning]]).

**Pourquoi un essai plutôt qu'un free tier seul pour Essentiel/Cercle** :
le coût d'activation est élevé (connecter 6 sources : agenda, mails, 2
WhatsApp, SMS, notes). Un free tier restreint ne peut pas démontrer la
valeur complète du produit avant que l'utilisateur ait fini de tout
connecter. Recommandation : essai de 14 jours avec accès total plutôt que
freemium seul, pour que l'aversion à la perte joue une fois les connexions
faites.

**Autres recommandations** : annuel prévu dès le lancement (badge grisé,
même non activé) ; preuve sociale dès le lancement via compteur de liste
d'attente (cohérent avec l'accès invite-only) ; éviter un pricing à l'usage
(nombre de messages agrégés) — préférer un forfait par tier, plus lisible
pour un usage familial partagé.
